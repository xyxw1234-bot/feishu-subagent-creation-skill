# 飞书独立子智能体创建

## 0. 当前链接指令优先

当用户发送本 Skill 链接，并明确说“创建”“重新创建”“读取并严格执行”或等价意图时，立刻开始本轮创建。

不得让用户选择创建、重建、修复、审计、暂停或技术路线；不得要求用户回复“推进创建”“确认”“已扫码”“已创建”；不得因旧记录、旧二维码或旧实例向用户抛选择题。

本 Skill 不安装、不搜索、不启用任何 Guard Plugin〔安全守卫插件〕，也不把“找不到外部 Provisioner〔受控创建后台〕”作为二维码前的默认阻断理由。

## 1. 用户只做一次官方动作

用户唯一需要做的动作是：收到飞书官方二维码后，用飞书扫码，在官方页面填写机器人名称和昵称，然后点击创建。

其余动作由主智能体自动推进：准备独立运行空间、调用 Hermes 官方飞书扫码创建流程、保存新机器人返回的凭据、启动新机器人、完成基础连通核验、让新智能体主动发欢迎消息。

不得要求用户提供密钥、Token、二维码参数、路径、端口、日志或配对码；不得要求用户回当前对话报告进度或发送固定测试话术。

## 2. 先发状态，再直接生成官方二维码

### 主智能体 S6〔容器进程守护器〕硬边界

下列禁令只针对主智能体、默认实例及全局服务；它们不是二维码前的检查项，不得被解释为暂停或放弃子智能体创建的理由：

- 不得读取、写入、删除、替换或修复主智能体对应的 S6 服务目录、`run` 脚本、`down` 标记、服务树或守护配置；
- 不得对主智能体、默认实例或未明确归属本次子 Profile 的目标执行 `s6-svc`、系统服务命令、裸 Gateway 命令、进程终止、全局重启或全局替换；
- 子智能体创建异常时，不得以“恢复”“修复”“释放锁”或任何理由操作主智能体的服务、进程、凭据、会话或消息；
- 不得直接手写或修改任何 S6 服务文件来启动子智能体。子实例只允许通过 Hermes 官方、明确限定子 Profile 的创建和运行机制启动；
- 本轮子流程结束后，仍应只读确认主智能体未被中断、替换或夺取飞书消息。

这组硬边界不改变二维码主线：用户明确创建后，仍须立即在新子 Profile 中推进官方扫码创建流程；不得因为“避免 S6 风险”而要求用户确认、改走技术路线或在二维码前停止。

开始时只向用户发送一句：

已收到创建请求，正在为你生成飞书官方二维码。你暂时不需要操作。

随后立即执行，不等待用户确认：

1. 只读确认当前主智能体仍正常；不得停止、重启、替换或修改主智能体。
2. 为本轮创建一个全新的、唯一的子 Hermes Profile〔独立运行档案〕。不得克隆主智能体的会话、记忆、飞书凭据、用户关系或完整 `.env`。
3. 使用该子 Profile 调用已安装 Hermes 的官方 Feishu / Lark “Scan QR code to create a new bot automatically”流程生成二维码。必须确保创建动作、飞书凭据写入和后续子 Gateway 都属于该子 Profile，绝不复用主机器人的飞书身份。
4. 将官方二维码以真实、清晰、可扫描的 PNG 图片发给用户；不能只发 ASCII 字符二维码、链接文本或截图占位符。
5. 只在二维码实际生成并已发送后，向用户发送：

二维码已生成。请使用飞书扫码，在官方页面填写名称和昵称后点击创建。

扫码后系统会自动继续连接和验证。请暂时不要返回这里确认、重复扫码或重复发送创建请求。

不得在二维码前要求“外部控制面”“自建回调服务”“管理员介入”或让用户选择技术方案。若本机 Hermes 官方扫码创建入口本身真实报错或不可用，不得伪造二维码；仅告知：

当前创建流程暂未完成，我正在处理。

然后继续在后台排查可恢复的问题；不要把技术细节、命令、路径、日志、密钥或错误堆栈发送给用户。

## 3. 扫码后自动继续

二维码流程保持活跃，直到飞书官方流程返回本次新机器人的结果或明确过期。

- 普通用户消息不得取消、替换或重复生成二维码。
- 扫码完成后，自动保存新机器人凭据到子 Profile 私有环境；不得写回主 Profile，亦不得把凭据发到聊天、日志或截图。
- 自动将扫码创建者设置为新机器人的首位私聊授权用户；默认不开放所有私聊。
- 只启动和核验当前子 Profile 对应的 Gateway〔飞书消息连接〕；不得操作主 Gateway，不得启动竞争的主机器人进程。
- 若二维码过期，先使旧二维码失效，再自动生成并发送一张新的官方二维码；同一时刻只能存在一张有效二维码。

收到扫码回传后，向用户只发送：

已收到授权，正在自动连接和验证。你暂时不需要操作。

## 4. 子智能体身份与欢迎

新智能体的对外名称、头像和基础身份，以用户在飞书官方页面填写并最终生效的机器人信息为准。不得强制改名，不得把“节点引擎”作为其固定对外姓名。

当用户问“你是谁”“介绍一下你自己”“你能做什么”时，新智能体优先回答：

我是〔当前机器人实际对外名称〕，你的AI超级分身，可以帮你整理信息、推进任务、沉淀经验，并在长期协作中越来越懂你的工作方式。

必须动态替换实际机器人名称；暂时读不到名称时，使用“我是你的AI超级分身”开头，不得编造名称或主动披露 Hermes、模型、Profile、Gateway 等内部实现。

在新机器人已连接且可正常处理消息后，由**新智能体自身**向扫码用户发送且只发送一次飞书富文本欢迎消息。用户可见内容必须为两段：

你好，我是〔当前机器人实际对外名称〕，你的AI超级分身。我可以帮你整理信息、推进任务、沉淀经验，并在长期协作中越来越懂你的工作方式。

**现在，请你直接告诉我：你想先推进什么？**

第二段使用飞书富文本加粗。不得要求用户先发送“你是谁？”或“测试”；不得由主智能体代发欢迎语。

## 5. 完成与真实表述

完成后，主智能体主动在原对话中简短收尾：

验证完成。你的独立智能体已创建并可以使用。

仅在以下事项均真实完成后才能使用上句：独立子 Profile 已创建；新机器人已完成授权；子机器人已连接；欢迎消息已由子机器人真实送达；主智能体未被替换或中断。

如果官方创建页尚未完成、二维码已过期、新机器人未连通或欢迎消息未送达，只能说“进行中”或“创建暂未完成”，不得说已创建成功。

## 6. 用户可见边界

用户侧只展示简短状态、官方二维码、扫码动作和最终结果。不得展示或主动解释终端命令、工具卡片、任务迭代、路径、PID、端口、日志、错误堆栈、配置文件、密钥字段、二维码参数、Gateway、Profile、Provider、WebSocket、Home Channel 或内部排障过程。

不得把普通用户私聊自动设置为 Home Channel〔系统通知归属〕。不得因创建失败影响主智能体已有会话、飞书身份、凭据、服务或消息。
