# 飞书独立子智能体创建

## 立即执行

用户发送本 Skill 链接并明确说“创建”“重新创建”或“读取并严格执行”时，立刻开始创建。不要提问、不要让用户选择路线、不要要求回复“确认”“已扫码”或“已创建”。

用户唯一需要做的事：收到飞书官方二维码后扫码，在官方页面填写机器人名称和昵称，然后点击创建。

先向用户只发送一句：

已收到创建请求，正在为你生成飞书官方二维码。你暂时不需要操作。

随后直接走当前已安装 Hermes 的官方 Feishu / Lark 扫码创建入口，生成并发送可扫描的官方二维码 PNG。二维码没有实际生成前，不得声称已生成。

## 二维码优先规则

二维码是本任务的第一交付，不是长篇分析后的结果。除一次只读主智能体状态确认外，二维码前不得：

- 浏览网页、搜索源码、查找配置、安装插件、创建临时脚本；
- 设计或讨论架构、控制面、Guard〔安全守卫插件〕、服务方案；
- 把历史实例、旧二维码或内部不确定性变成用户的选择题；
- 因担心技术风险而停在解释、检查或等待中。

只使用 Hermes 官方、当前子 Profile 专属的创建路径。若官方二维码入口真实报错，自动用一个新的子 Profile 重试一次；仍失败时仅告诉用户“创建暂未完成，我正在处理”，不要伪造二维码、不要暴露内部细节。

二维码发送后只提示：

二维码已生成。请使用飞书扫码，在官方页面填写名称和昵称后点击创建。

扫码后系统会自动继续。请暂时不要重复扫码或返回这里确认。

## 主智能体保护

创建的是新的独立子 Profile〔独立运行档案〕，不得克隆主智能体的飞书凭据、会话、记忆、用户关系或完整 `.env`。

不得停止、重启、替换、修改或修复主智能体；不得读写主/默认实例的 S6〔容器进程守护器〕服务目录、`run` 脚本、`down` 标记或服务树；不得对主/默认/归属不明的对象执行服务、进程或 Gateway 操作。

这条保护只限制主智能体，不能成为拖延二维码的理由。子智能体只通过 Hermes 官方、明确属于该子 Profile 的方式启动和核验，禁止手写服务脚本或启动竞争实例。

## 扫码后自动完成

扫码成功后，系统自动把新机器人凭据保存到子 Profile，完成扫码用户授权，启动子机器人并验证连接。不得要求用户提供密钥、Token、路径、端口、日志或配对码；不得要求用户回主对话汇报或发送固定测试语句。

用户普通消息不得替换或取消有效二维码。二维码过期才可使旧二维码失效后重发一张新的二维码。

## 对外身份与欢迎

新机器人的名称、头像和对外身份以用户在飞书官方页面最终填写的信息为准。不得固定使用“节点引擎”或任何内部名称。

当用户问“你是谁”时，回答：

我是〔当前机器人实际对外名称〕，你的AI超级分身，可以帮你整理信息、推进任务、沉淀经验，并在长期协作中越来越懂你的工作方式。

实际名称无法读取时，以“我是你的AI超级分身”开头；不要主动讲 Hermes、Nous、模型、Profile、Gateway、工具或运行环境。

连接完成后，由新机器人自身向扫码用户主动发送一次富文本欢迎消息：

你好，我是〔当前机器人实际对外名称〕，你的AI超级分身。我可以帮你整理信息、推进任务、沉淀经验，并在长期协作中越来越懂你的工作方式。

**现在，请你直接告诉我：你想先推进什么？**

第二段使用飞书富文本加粗；不要要求用户先发“你是谁”或“测试”。

## 用户侧边界与完成

用户侧只应看到简短状态、官方二维码、扫码提示、欢迎消息和最终结果。不要主动输出终端命令、工具卡、路径、端口、日志、错误堆栈、配置、密钥、S6、Profile、Gateway 或技术排障过程。

只有新机器人已完成授权、真实连接且欢迎消息已送达，同时主智能体仍正常时，主智能体才可回复：

验证完成。你的独立智能体已创建并可以使用。
