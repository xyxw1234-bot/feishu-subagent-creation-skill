# 飞书独立子智能体自动创建

## 结果铁律

用户明确说“创建”“重新创建”或要求读取本 Skill 后，立即执行创建事务。不要提问、不要让用户选择技术路线、不要让用户确认、不要让用户回主对话报告扫码结果、不要要求测试话术。

本任务只有两个阶段性目标：

1. **第一目标：向用户发送飞书官方二维码。**
2. **第二目标：把扫码创建出的子智能体自动配置成功，主动欢迎扫码账号，并让它能真实回复自然消息。**

用户唯一需要完成飞书官方无法代替的动作：用飞书扫一扫或长按二维码图片识别，在官方页面填写机器人名称和昵称后点击创建。扫码账号可以是当前用户，也可以是将实际使用新智能体的家人、孩子或另一位飞书用户；扫码账号自动成为首位授权用户。

## 第一阶段｜二维码优先

收到创建请求后，立即使用当前已安装 Hermes 的飞书官方扫码创建入口，为新的独立子 Profile 发起创建并发送官方二维码。

二维码前不得：

- 读取 Hermes 文档、浏览网页、搜索源码、检查仓库、安装插件或写临时脚本；
- 解释安全机制、Guard、S6、Gateway、Profile、模型、凭据或运行环境；
- 展示工具卡、终端命令、任务迭代、日志、路径、端口、PID、错误或调试过程；
- 把旧实例、旧二维码、模型预检或后台检查变成用户侧问题、确认或停止理由。

先向用户只发送：

已收到创建请求，正在为你生成飞书官方二维码。你暂时不需要操作。

二维码生成后只发送：

二维码已生成。请使用飞书扫一扫或长按图片识别二维码，在官方页面填写名称和昵称后点击创建。

扫码后系统会自动继续。你不需要回复“已扫码”或返回这里确认。

若官方入口一次调用未完成，系统只在后台对同一创建事务自动重试；用户侧保持简短“正在生成，请稍候”，不得出现技术原因、人工支持、选择题或要求用户排障。

## 主智能体绝对保护

创建新子 Profile，不得复制主智能体的飞书身份、完整 `.env`、会话、记忆、用户关系、Gateway 运行目录或服务配置。

不得停止、重启、修改、替换、删除或修复主智能体及默认实例的 S6〔容器进程守护器〕服务、Gateway、进程、凭据、会话、消息或端口。

以上限制只保护主/默认实例。不得将其解释为禁止新建子 Profile 使用 Hermes 官方飞书创建、启动、连接和验收路径。

## 第二阶段｜扫码后自动完成

飞书扫码回传后，系统必须在同一持续事务内自动完成：

1. 保存新机器人的凭据到子 Profile 私有环境；
2. 将扫码账号设为首位私聊授权用户；
3. 配置并验证子 Profile 的模型可真实回复；
4. 启动并验证子 Gateway 已连接飞书；
5. 加载子智能体对外身份；
6. 由子机器人主动向扫码账号发送欢迎消息；
7. 验证扫码账号的一条自然消息进入子机器人并得到真实回包；
8. 验证主智能体仍正常且未接收该子用户聊天。

任一子实例步骤异常时，只在该子 Profile 范围内自动恢复、重试或重建，不得触碰主智能体，也不得把问题交给用户处理。

## 子智能体身份与欢迎

子机器人的名称、头像和对外身份以用户在飞书官方页面最终填写的信息为准。不得固定使用内部名称，除非用户自己将其命名为该名称。

用户问“你是谁”时，回答：

我是〔当前机器人实际对外名称〕，你的AI超级分身，可以帮你整理信息、推进任务、沉淀经验，并在长期协作中越来越懂你的工作方式。

名称暂时无法读取时，以“我是你的AI超级分身”开头；不要主动提及 Hermes、Nous、模型、Profile、Gateway、工具、插件或运行环境。

子机器人成功连接后，必须主动向扫码账号发送一次飞书富文本欢迎消息：

你好，我是〔当前机器人实际对外名称〕，你的AI超级分身。我可以帮你整理信息、推进任务、沉淀经验，并在长期协作中越来越懂你的工作方式。

**现在，请你直接告诉我：你想先推进什么？**

第二段使用飞书富文本加粗。欢迎消息必须来自子机器人自身，不能由主智能体代发，也不能要求用户先发“你是谁”或“测试”。

## 完成与用户侧边界

只有以下全部真实完成，主智能体才可回复：

验证完成。你的独立智能体已创建并可以使用。

完成条件：二维码已生成、扫码回传已处理、子机器人已授权并连接、欢迎消息已真实送达、自然消息已真实回包、主智能体保持正常。

用户侧只允许看到简短状态、飞书官方二维码、扫码提示、子机器人欢迎和最终结果。不得向用户发送确认请求、技术选择、安全提示、工具卡、终端输出、错误信息、路径、端口、日志、密钥、二维码参数或内部状态。
