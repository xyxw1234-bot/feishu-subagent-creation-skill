# Changelog

## v1.1.0

- 基于 `v1.0.8` 恢复仓库主页作为人类与智能体共用分发入口、唯一 `SKILL.md` 执行源、可直接转发文案、免测试用户流程与固定 Release 入口；
- 移除 `plugin/` 中会注册 `pre_tool_call` 的 Guard Plugin〔安全守卫插件〕，不再将命令拦截器置入用户分发主链；
- 保持干净根目录：仅保留 Skill、说明、许可证与版本记录。

## v1.0.9

- 首次清理公开分发主链中的 Guard Plugin 文件与重复 Skill 目录；
- GitHub 主页仅保留用户需要的 Skill、说明、许可证与版本记录。

## v1.0.7

- 保持轻量 Guard Plugin：只拦截已知主智能体风险操作，不新增控制面或事务后端。
- 修正子 Gateway 恢复验收：不再要求扫码用户发送“你是谁？”、“测试”或回主对话确认。
- 明确 Profile 独立不是操作系统级隔离，且欢迎介绍仅可承诺已启用、已授权能力。
- 保留子智能体主动欢迎介绍的正文、标题和富文本排版规则。

## v1.0.6

- 优化子智能体第一条主动欢迎消息，增加自然问候与正式介绍；
- 固化“节点引擎｜时代先行者的AI超级分身”主标题及两层能力介绍；
- 强制使用飞书富文本 Post 消息、真实加粗和段落排版，禁止原始 Markdown 格式源码；
- 保留欢迎消息一次发送、事务幂等、真实送达和不要求用户测试的规则。

## v1.0.5

- 子智能体 READY 后主动向扫码用户发送一次简短欢迎介绍；
- 用户不再需要发送“你是谁？”或任何固定测试话术；
- 欢迎消息纳入幂等、真实送达和出站能力验收；
- 明确区分欢迎消息送达与用户后续自然消息的入站验证。

## v1.0.4

- 当前链接指令优先：新链接 + 明确执行意图直接启动，不受历史测试记录、旧菜单或旧任务状态干扰；
- 禁止创建流程中调用澄清/选择工具或要求用户回复数字、选项文字、确认口令；
- 增加用户可见状态文案：清楚说明系统当前阶段、用户是否需要等待或操作；
- 扫码后明确提示系统会自动继续，不要求回当前对话确认；
- 重启恢复验证改为后台自动验收，不再要求用户做第二次测试或回主对话确认。

## v1.0.3

- 修复正式 Skill 正文中误带的行号前缀，恢复可读的纯文本提示词；
- 根目录与中文目录的 `SKILL.md` 改为逐字一致；
- Guard Plugin 增加相对路径 `.env` 的拦截；
- 对齐 Skill 与插件的版本号。

## v1.0.2

- 在仓库根目录增加兼容性安装入口 `SKILL.md`；
- 保留中文目录版本，并在 README 中使用 URL 编码的中文路径；
- 修复部分聊天客户端截断中文 Raw GitHub 路径的问题。

## v1.0.1

- 固定 README 中的安装链接到不可变 Release 标签；
- 完成从 GitHub 全新安装、启用轻量 Guard Plugin 和中文 Skill 目录验证。

## v1.0.0

- 自动推进到飞书二维码图片；
