# Changelog

## v1.0.4

- 当前链接指令优先：新链接 + 明确执行意图直接启动，不受历史测试记录、旧菜单或旧任务状态干扰；
- 禁止创建流程中调用澄清/选择工具或要求用户回复数字、选项文字、确认口令；
- 增加用户可见状态文案：清楚说明系统当前阶段、用户是否需要等待或操作；
- 扫码后明确提示系统会自动继续，不要求回当前对话确认；
- 重启恢复验证改为后台自动验收，不再要求用户做第二次测试或回主对话确认。

## v1.0.3

- 修复正式 Skill 正文中误带的行号前缀，恢复可读的纯文本提示词；
- 根目录与中文目录的 `SKILL.md` 改为逐字一致；
- Guard Plugin 增加相对路径 `.env` 的拦截；
- 对齐 Skill 与插件的版本号。

## v1.0.2

- 在仓库根目录增加兼容性安装入口 `SKILL.md`；
- 保留中文目录版本，并在 README 中使用 URL 编码的中文路径；
- 修复部分聊天客户端截断中文 Raw GitHub 路径的问题。

## v1.0.1

- 固定 README 中的安装链接到不可变 Release 标签；
- 完成从 GitHub 全新安装、启用轻量 Guard Plugin 和中文 Skill 目录验证。

## v1.0.0

- 自动推进到飞书二维码图片；
- 禁止用户选择技术路线、回复“推进创建”或“已创建”；
- 禁止主动输出技术过程、工具记录、日志与复杂技术报告；
- 固化子智能体身份、二维码事务、用户授权、真实回包、重启恢复和完成表述规则。
