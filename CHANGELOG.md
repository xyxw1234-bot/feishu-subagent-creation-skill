# Changelog

## v1.2.1

- 新增主智能体 S6〔容器进程守护器〕硬边界：禁止直接读写或修复主服务目录、`run` 脚本、`down` 标记、服务树和守护配置；禁止对主/默认/未明确归属子 Profile 的目标执行 `s6-svc`、全局服务、裸 Gateway、进程终止或全局重启；
- 明确该边界不构成二维码前置阻断：用户明确创建后，必须继续在新子 Profile 内走官方扫码创建流程；
- 全文复核并调整执行顺序，确保 S6 边界在二维码主线前生效，且不打断“生成二维码 → 扫码 → 自动连接 → 主动欢迎”的用户路径。

## v1.2.0

- 删除 Skill 本文中“缺少外部受控创建后台即二维码前安全停止”的硬阻断路径；
- 固定为：用户明确创建后，主智能体直接调用已安装 Hermes 的官方飞书扫码创建入口，先生成并发送官方二维码；
- 保留主智能体保护、独立子 Profile、新机器人凭据不回写主 Profile、扫码用户首位授权、子机器人主动欢迎和真实完成表述；
- 明确本仓库不安装、搜索或启用 Guard Plugin，不能再把 Guard 或不存在的外部 Provisioner 当作默认创建前置条件。

## v1.1.4

- 子智能体的对外名称、头像和身份以用户在飞书官方页面设置的实际机器人信息为准；
- “节点引擎”仅作为系统产品名称，不再作为所有子智能体固定对外名称；
- “你是谁”回复、内部身份预检、欢迎富文本和最终验收同步使用用户自定义机器人身份。

## v1.1.3

- 不改动创建协议、欢迎文案、二维码、隔离或验收规则。

## v1.1.2
