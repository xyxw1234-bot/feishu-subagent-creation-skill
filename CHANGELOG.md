# Changelog

## v1.0.8.3

- 仅优化二维码交付文案：增加二维码图片未显示时的飞书官方链接兜底与“重发二维码”提示；
- 强调用户可用自己的飞书账号扫码，也可由家人、孩子等实际使用者扫码；扫码账号成为首位使用者并拥有自己的 AI 超级分身。

## v1.0.8.2

- 重写为“二维码第一目标、子智能体成功第二目标”的自动创建协议；
- 禁止二维码前的文档读取、源码搜索、仓库检查、插件安装、用户确认和技术解释；
- 删除 Guard 插件分发物，不再注册工具调用拦截；
- 将 S6 保护严格限定为主/默认实例，明确允许新子 Profile 的官方创建路径；
- 明确扫码回传后自动完成授权、模型、Gateway、主动欢迎、自然回包和主子隔离验证。

## v1.0.8.1

- 在 622c21d 完整执行链基础上增加“二维码第一交付”置顶规则；
- 禁止二维码前读取 Hermes 文档、搜索源码、检查仓库、解释 Guard 或做用户可见长时间环境审查；
- 删除用户可见“当前无法安全完成创建 / 安全停止”措辞，官方二维码入口失败只报告“二维码暂未生成，正在处理”；
- 将 S6 约束限定为主/默认实例保护，不得被解释为禁止新子 Profile 的官方创建路径。

## v1.0.8

- 仓库主页改为人类与智能体共用的正式分发入口。
- 根目录只保留唯一 `SKILL.md` 执行源；删除重复目录入口。
- 将可选轻量 Guard Plugin 收纳至 `plugin/`，避免首页出现无上下文代码文件。
- README 增加可直接转发给用户智能体的简短执行文案。

## v1.0.7

- 保持轻量 Guard Plugin：只拦截已知主智能体风险操作，不新增控制面或事务后端。
- 修正子 Gateway 恢复验收：不再要求扫码用户发送“你是谁？”、“测试”或回主对话确认。
- 明确 Profile 独立不是操作系统级隔离，且欢迎介绍仅可承诺已启用、已授权能力。
- 保留子智能体主动欢迎介绍的正文、标题和富文本排版规则。

## v1.0.6

- 优化子智能体第一条主动欢迎消息，增加自然问候与正式介绍；
- 固化“节点引擎｜时代先行者的AI超级分身”主标题及两层能力介绍；
- 强制使用飞书富文本 Post 消息、真实加粗和段落排版，禁止原始 Markdown 格式源码；
- 保留欢迎消息一次发送、事务幂等、真实送达和不要求用户测试的规则。

## v1.0.5

- 子智能体 READY 后主动向扫码用户发送一次简短欢迎介绍；
- 用户不再需要发送“你是谁？”或任何固定测试话术；
- 欢迎消息纳入幂等、真实送达和出站能力验收；
- 明确区分欢迎消息送达与用户后续自然消息的入站验证。

## v1.0.4
